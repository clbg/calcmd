## Architecture

The core library has a clean pipeline:

```
markdown string
  в†’ Parser         (produces Table with Row/Column/Cell structure)
  в†’ FormulaParser  (parses formula strings into Expression ASTs)
  в†’ Evaluator      (resolves dependencies, computes values, collects errors)
  в†’ ParsedTable    (Table + dependency graph + validation errors)
```

`website` consumes both `@calcmd/core` and `@calcmd/ui` as workspace dependencies.

## Key Conventions

- All types are defined in `types.ts` вЂ” don't scatter type definitions across files
- The public API surface is `index.ts` вЂ” only export what consumers need
- `calcmd(markdown: string): ParsedTable` is the main entry point
- Errors are collected and returned (non-throwing) вЂ” callers check `result.errors`
- The spec in `docs/04-Spec.md` is the source of truth; implementation follows it
- Run all commands from the repo root using pnpm
