## Architecture

The core library has a clean pipeline:

```
markdown string
  → Parser         (produces Table with Row/Column/Cell structure)
  → FormulaParser  (parses formula strings into Expression ASTs)
  → Evaluator      (resolves dependencies, computes values, collects errors)
  → ParsedTable    (Table + dependency graph + validation errors)
```

The playground consumes `@calcmd/core` as a workspace dependency (`workspace:*`) and renders the `ParsedTable` output.

## Key Conventions

- All types are defined in `types.ts` — don't scatter type definitions across files
- The public API surface is `index.ts` — only export what consumers need
- `calcmd(markdown: string): ParsedTable` is the main entry point
- Errors are collected and returned (non-throwing) — callers check `result.errors`
- The spec in `docs/04-Spec.md` is the source of truth; implementation follows it
- Run all commands from `poc/` (workspace root) using pnpm
