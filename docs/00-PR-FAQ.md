# PR/FAQ: CalcMD

## Press Release

**FOR IMMEDIATE RELEASE**

### CalcMD: The Missing Standard for Verifiable Tables in the AI Era

*Open-source format brings formula transparency to AI-generated markdown tables*

**March 2026** — Today marks the launch of CalcMD, an open specification that extends markdown tables with embedded formulas, solving a critical trust problem in AI-generated content.

**The Problem: AI Calculations You Can't Verify**

As AI assistants become our daily collaborators, they frequently generate tables with calculated values—expense splits, sales summaries, data analysis. But users face a dilemma: the numbers look authoritative, yet there's no way to verify the calculations without manually checking each cell or asking the AI again (consuming more tokens and time).

"I was splitting travel expenses with a friend using ChatGPT," says early adopter Charlie Peng. "The AI calculated who owed what, but I had no idea if the math was right. I ended up copying everything to Excel just to verify. That's when I knew we needed CalcMD."

**The Solution: Formulas in Plain Text**

CalcMD introduces a simple, human-readable syntax for embedding formulas directly in markdown tables:

```markdown
| Item   | Qty | Price | Total=Qty*Price |
|--------|-----|-------|-----------------|
| Ramen  | 2   | 1500  | 3000            |
| Ticket | 1   | 200   | 200             |
| **Sum** |     |       | **3200=sum(Total)** |
```

The formulas are visible, verifiable, and executable by tools—yet remain readable without any special software.

**Designed for the AI Era**

CalcMD is specifically optimized for AI workflows:
- **Easy to generate**: AI models can learn the syntax in their system prompt
- **Easy to verify**: Users can validate calculations with one click
- **Git-friendly**: Plain text format works perfectly with version control
- **Tool-supported**: Browser extensions, CLI tools, and language libraries

**Open Standard, Growing Ecosystem**

CalcMD launches with:
- Complete specification (v0.1)
- Reference implementations in JavaScript and Python
- Web-based playground for instant testing
- Browser extension for ChatGPT, Claude, and Gemini
- Integration guides for popular AI platforms

"We're not trying to replace Excel or Google Sheets," explains the CalcMD team. "We're solving a different problem: making lightweight calculations in markdown documents trustworthy and verifiable. This is especially critical in the AI era where unverifiable numbers can spread quickly."

**Path to Industry Standard**

The CalcMD specification is released under CC0 (public domain) to encourage adoption. The team is working with:
- Markdown parser maintainers (CommonMark, GFM)
- AI platform developers (OpenAI, Anthropic, Google)
- Developer tool vendors (GitHub, VS Code, JetBrains)
- Documentation platforms (GitBook, Docusaurus, Obsidian)

**Availability**

CalcMD is available today:
- Specification: github.com/calcmd/spec
- Playground: calcmd.dev
- Documentation: docs.calcmd.dev

**About CalcMD**

CalcMD is an open-source initiative to bring verifiable calculations to markdown tables. The project is community-driven and welcomes contributions from developers, technical writers, and anyone who values trustworthy data.

## Frequently Asked Questions

### General Questions

**Q: What is CalcMD?**
A: CalcMD (Calculated Markdown) is an open specification that extends markdown table syntax to include formulas. It makes calculated values verifiable and transparent.

**Q: Why does this matter?**
A: In the AI era, we're increasingly consuming tables generated by AI assistants. When these tables include calculations (sums, averages, products), there's currently no way to verify the math without manual checking or re-asking the AI. CalcMD solves this by embedding formulas directly in the table.

**Q: Who is this for?**
A: Three primary audiences:
1. **AI users** who want to verify calculations in ChatGPT/Claude responses
2. **Developers** who need Git-friendly tables with calculations
3. **Data analysts** who want lightweight computed tables without Excel

**Q: How is this different from Excel/Google Sheets?**
A: CalcMD is not a replacement for spreadsheets. It's designed for:
- Plain text workflows (Git, markdown editors)
- AI-generated content
- Simple calculations (not pivot tables, macros, charts)
- Human-readable formulas (no A1/B2 references)

**Q: How is this different from Jupyter Notebooks?**
A: Jupyter is for executable code and complex analysis. CalcMD is for simple tabular calculations embedded in documents. Think of it as "markdown tables + basic formulas" rather than "computational notebook."

### Technical Questions

**Q: What formula syntax do you support?**
A: CalcMD v0.1 supports:
- Arithmetic: `+`, `-`, `*`, `/`, `%`, `()`
- Functions: `sum()`, `avg()`, `min()`, `max()`, `count()`, `round()`
- Column references: use column names (e.g., `Price*Qty`)
- Row references: implicit (current row)
- Conditional: `if(condition, true_val, false_val)`

**Q: Can I reference other cells like Excel's A1 notation?**
A: No, and that's intentional. CalcMD uses column-based formulas for clarity and Git-friendliness. Instead of `=A2*B2`, you write `=Qty*Price`.

**Q: Does CalcMD require special tools?**
A: No. CalcMD tables are readable in any markdown viewer—formulas appear as text. But tools enhance the experience:
- Web playground: instant verification
- Browser extensions: auto-verify in AI chats
- CLI tools: batch processing
- Editor plugins: live calculation

**Q: What if my markdown viewer doesn't support CalcMD?**
A: The table still renders normally. Formulas appear as text in the cells. CalcMD is designed for graceful degradation.

**Q: Is CalcMD secure?**
A: Yes. Unlike Excel formulas (which can execute arbitrary code), CalcMD:
- Only supports a whitelist of safe functions
- No file/network access
- No loops or recursion
- Maximum expression complexity enforced
- All implementations must sandbox evaluation

### Business & Strategy Questions

**Q: Is this a company or a standard?**
A: CalcMD is an open standard (like Markdown, TOML, JSON5). It's not owned by any company. We may form a non-profit foundation if/when needed for governance.

**Q: How will CalcMD become an industry standard?**
A: Path to standardization:
1. **Adoption**: Developers start using it (via tools)
2. **Ecosystem**: Multiple implementations and tools emerge
3. **Platform integration**: Major platforms add native support
4. **Formalization**: Submit to W3C/WHATWG or form independent standard body

**Q: What's the business model?**
A: There isn't one. The specification is public domain (CC0). Anyone can:
- Build tools and charge for them
- Integrate CalcMD in commercial products
- Offer consulting/training

The core team may eventually offer:
- Hosted validation API (for high-volume use)
- Enterprise support contracts
- But the spec remains free forever

**Q: Why should AI companies adopt this?**
A: Benefits for AI platforms:
1. **Reduces hallucination impact**: Even if AI makes calculation errors, users can catch them
2. **Improves trust**: Transparent formulas increase confidence
3. **Fewer re-queries**: Users don't need to ask "is this correct?" repeatedly
4. **Competitive advantage**: First platform with verifiable tables stands out

**Q: What if someone forks CalcMD and makes incompatible changes?**
A: We accept this risk. Our strategy:
1. Build the best reference implementation
2. Provide comprehensive test suite
3. Form community governance early
4. Achieve adoption momentum before forks become attractive

### Comparison Questions

**Q: How does CalcMD compare to Org-mode tables?**
A: Org-mode's table formulas are excellent but:
- Only work in Emacs
- Use Calc syntax (steeper learning curve)
- Not designed for AI interoperability

CalcMD is:
- Editor-agnostic
- AI-friendly syntax
- Simpler function set

**Q: What about DuckDB for CSV analysis?**
A: DuckDB is powerful but serves different needs:
- DuckDB: Serious data analysis, SQL queries, large datasets
- CalcMD: Lightweight tables in documents, human-readable formulas

They're complementary. You might export CalcMD to CSV → analyze in DuckDB.

**Q: How does this relate to literate programming (Quarto, Jupyter)?**
A: CalcMD is a subset:
- Literate programming: Mix prose, code, results (full programming environment)
- CalcMD: Just tables with formulas (no programming language needed)

Use CalcMD when you want simple calculations without the overhead of a computational notebook.

## External FAQ (anticipated questions from press/community)

**Q: Isn't this just reinventing CSV with formulas?**
A: No, CalcMD is about **human-readable tables in documents**, not data interchange. CSV is great for data transfer; CalcMD is for collaborative docs, AI outputs, and version control.

**Q: Why not just use Excel Online links?**
A: Excel Online is powerful but:
- Requires Microsoft account
- Binary format (not Git-friendly)
- Heavyweight for simple calculations
- Not embeddable in markdown docs

**Q: This seems like a solution looking for a problem.**
A: We've identified three concrete pain points:
1. Verifying AI-generated calculations (saves time and reduces errors)
2. Maintaining calculated tables in Git repos (current solution: commit generated results, lose formulas)
3. Lightweight collaboration on shared expenses / simple budgets (current solution: copy to Sheets, friction increases)

Each alone might not justify a new format, but together—especially with AI adoption growing—they form a compelling case.

**Q: Won't AI just get better and stop making calculation errors?**
A: Even when AI is more reliable:
1. Humans still make errors (CalcMD helps human-written tables too)
2. Transparency matters (you want to see HOW it calculated, not just trust the result)
3. Verification should be easy, not require trust

It's like checksums: even reliable systems benefit from verifiability.
